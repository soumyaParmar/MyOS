# Feature Planning Workspace

This folder stores feature-by-feature planning before code is written.

Workflow:

1. Choose the next unchecked checkbox from `../MyOS_TODO.md`.
2. Create one folder for that feature.
3. Add a `FEATURE_PLAN.md` using `FEATURE_PLAN_TEMPLATE.md`.
4. Let the user review and verify the plan.
5. Build the feature only after approval.
6. Mark the checkbox in `../MyOS_TODO.md` only after planning, implementation, and verification are complete.

Agents should keep one feature per folder and avoid combining multiple TODO checkboxes into one implementation unless the user explicitly approves it.
